**Data Flow Diagram (DFD)**

1. Draw a Context Diagram (Level 0) and a Level-1 Data Flow Diagram for an online bookstore. A customer searches for books, adds them to a shopping cart, and proceeds to checkout. The system verifies the availability of books with the warehouse inventory and processes the payment using a third-party bank gateway. Upon successful payment, a receipt is generated for the customer, and a dispatch request is sent to the warehouse.
2. Draw a Level 0 and a Level-1 Data Flow Diagram for a library management system. The system must handle processes like registering a new student, issuing books, returning books, and calculating late fines. External entities include Students, Librarians, and the University's Central Database (to verify student enrollment). Ensure you show data stores for Books, Members, and Transactions.
3. Draw a Level 0 and a Level-1 DFD for a food delivery platform. Customers browse restaurant menus and place food orders. The system routes the order details to the respective Restaurant and assigns an available Delivery Driver. Include processes for payment handling and order tracking, showing how data flows between the Customer, Restaurant, Driver, and the central system.

**Activity Diagram**

1. Draw an Activity Diagram illustrating the process of withdrawing cash from an ATM. The workflow should cover inserting the card, entering the PIN, validating the PIN (include a branch for an incorrect PIN), selecting the withdrawal amount, checking the account balance, dispensing cash, and printing a receipt.
2. Create an Activity Diagram representing an online checkout process. The flow begins when the user clicks "Checkout". The user must log in or continue as a guest. Then, they provide shipping details and select a payment method. Include a decision node for payment authorization (Success vs. Failure). Use a fork/join (parallel execution) to simultaneously send a confirmation email and deduct the item from the inventory after a successful payment.
3. Model a hotel room reservation process using an Activity Diagram. The process starts with a user searching for rooms by date. If rooms are available, the user selects a room and enters guest details. The payment is processed. If the payment fails, the user is prompted to try again. If successful, the system confirms the booking. Include swimlanes to separate the actions of the User, the Booking System, and the Payment Gateway.

**Class Diagram**

1. Draw a Class Diagram for a simplified e-learning platform. Key entities include `Student`, `Instructor`, `Course`, and `Module`. An `Instructor` can create multiple `Courses`, and a `Course` contains multiple `Modules`. A `Student` can enroll in many `Courses`. Include appropriate attributes (e.g., `studentId`, `courseName`), operations (e.g., `enroll()`, `addModule()`), and specify the multiplicities and relationships (such as aggregation/composition).
2. Create a Class Diagram for a hospital system. The system needs to keep track of `Doctors`, `Patients`, `Appointments`, and `MedicalRecords`. A `Doctor` has attributes like `specialty` and `schedule`, while a `Patient` has `contactInfo` and `medicalHistory`. Model the relationships to show that a patient can book multiple appointments, an appointment is handled by one doctor, and a patient has one medical record.
3. Design a Class Diagram for a ride-sharing application. The model should include classes for `Rider`, `Driver`, `Vehicle`, `RideRequest`, and `Payment`. Ensure relationships show that a `Driver` owns/drives a `Vehicle`, a `Rider` creates a `RideRequest`, and a completed ride generates a `Payment`. Include visibility modifiers (`+`, `-`, `#`) for class attributes and methods.
